# Plots Lx/Tx as a function of the regenerative dose (DEPRECATED)

This function plots `Lx/Tx` values as a function of regenerative dose,
for every selected aliquot and for each sample.

## Usage

``` r
LT_RegenDose(
  DATA,
  Path,
  FolderNames,
  SampleNames = FolderNames,
  Nb_sample,
  BinPerSample = rep(1, Nb_sample),
  SG = rep(TRUE, Nb_sample),
  sepDP = c(","),
  nrow = 3L,
  ncol = nrow
)
```

## Arguments

- DATA:

  [list](https://rdrr.io/r/base/list.html) (**required**): list of
  objects `LT`, `sLT`, `ITimes`, `dLab`, `ddot_env`, `regDose`, `J`,
  `K`, `Nb_measurement`, \#' provided by
  [Generate_DataFile](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile-deprecated.md)
  or
  [Generate_DataFile_MG](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile_MG-deprecated.md)
  or
  [combine_DataFiles](https://crp2a.github.io/BayLum/dev/reference/combine_DataFiles.md).`DATA`
  can contain information from more than one sample.

- Path:

  [character](https://rdrr.io/r/base/character.html) (**required**):
  path to the project folder (the same as the one used in
  [Generate_DataFile](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile-deprecated.md)
  or
  [Generate_DataFile_MG](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile_MG-deprecated.md)
  to provide `DATA`)

- FolderNames:

  [character](https://rdrr.io/r/base/character.html) (**required**):
  vector of names of the sub-folders containing the BIN-files, which
  were used by
  [Generate_DataFile](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile-deprecated.md)
  or
  [Generate_DataFile_MG](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile_MG-deprecated.md)
  to generate the `DATA` object.

- SampleNames:

  [character](https://rdrr.io/r/base/character.html) (with default):
  Names of samples. To use if there is more than one bin file per
  sample.

- Nb_sample:

  [integer](https://rdrr.io/r/base/integer.html) (**required**): ID
  number (in `[1,Nb_sample]`) of the sample selected for plotting L/T as
  a function of regenerative doses. Required if the `DATA` object
  contains information for more than one sample.

- BinPerSample:

  [integer](https://rdrr.io/r/base/integer.html) (with default): integer
  vector (with default): vector with the number of BIN files per sample,
  which was used in
  [Generate_DataFile](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile-deprecated.md)
  or
  [Generate_DataFile_MG](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile_MG-deprecated.md)
  to generate the `DATA` object.

- SG:

  [logical](https://rdrr.io/r/base/logical.html) (with default): vector
  to set the type of measurement for each sample
  `length(SG)=Nb_sample`.If the sample of number ID equal to `i`,
  `SG[i]=TRUE` if it is a Single-grain OSL measurements, `SG[i]=FALSE`
  if it is a Multi-grain OSL measurements.

- sepDP:

  [character](https://rdrr.io/r/base/character.html) (with default):
  column separator in the `DiscPos.csv` file or in Disc.csv file. It
  must be the same separator for all samples, for single-grain OSL
  measurements or multi-grain OSL measurements.

- nrow:

  [integer](https://rdrr.io/r/base/integer.html) (with default):
  controls the arrangement of the plots, here the number of rows. Can be
  set to `NULL`.

- ncol:

  [integer](https://rdrr.io/r/base/integer.html) (with default):
  controls the arrangement of the plots, here the number of columns. Can
  be set to `NULL`.

## Value

Lx/Tx plots; there are as many plots as selected aliquots in the
`DiscPos.csv` file. There are 9 plots per page. There is not
interpolation.

## Details

To fill `FolderNames` and `BinPerSample`, we refer to the **Detail**
section from the
[Generate_DataFile](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile-deprecated.md)

[Generate_DataFile_MG](https://crp2a.github.io/BayLum/dev/reference/Generate_DataFile_MG-deprecated.md)
function. As well for a precise description of input `DATA`.
